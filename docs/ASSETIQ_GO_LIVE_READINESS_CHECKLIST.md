# AssetIQ Go-Live Readiness Checklist

Concise, executive-friendly checklist for implementation teams and customers to confirm readiness before AssetIQ go-live. Use alongside the [Client Onboarding Playbook](./CLIENT_ONBOARDING_PLAYBOOK.md) and **Settings → Success Readiness** for ongoing measurement.

---

## 1. Project

- [ ] Executive Sponsor assigned
- [ ] Project Manager assigned
- [ ] Change Lead assigned
- [ ] Go-live date agreed
- [ ] Success criteria agreed

---

## 2. People

- [ ] Users imported
- [ ] Roles assigned
- [ ] Key users trained
- [ ] AssetIQ Champions appointed
- [ ] End-user training completed

---

## 3. Equipment Data

- [ ] Equipment hierarchy imported
- [ ] Equipment criticality completed
- [ ] Failure modes imported or created
- [ ] Maintenance strategies imported
- [ ] Core asset data validated

---

## 4. Processes

- [ ] Observation workflow agreed
- [ ] Maintenance workflow configured
- [ ] Digital forms created
- [ ] SOPs updated
- [ ] Daily work process agreed

---

## 5. Technology

- [ ] Tenant configured
- [ ] Modules enabled
- [ ] User permissions configured
- [ ] Mobile devices tested
- [ ] Wi-Fi / Internet verified

---

## 6. Testing

- [ ] User Acceptance Testing completed
- [ ] Mobile testing completed
- [ ] Workflow testing completed
- [ ] User sign-off received
- [ ] Go-live approved

---

## Required Customer Resources

| Role | Purpose |
|------|---------|
| Executive Sponsor | Executive accountability and go-live approval |
| Project Manager | Implementation planning and delivery |
| Change Lead | Adoption, training, and change management |
| Maintenance Representative | Maintenance data, workflows, and PM programs |
| Operations Representative | Daily operations and field usage |
| IT Representative | Tenant, access, mobile, and connectivity |
| AssetIQ Champion(s) | Local support and adoption in each area |

---

## Required Customer Documentation

Provide or confirm availability of:

- Equipment hierarchy
- User list
- Existing PM program
- Existing failure modes or FMEA (if available)
- Standard Operating Procedures (SOPs)
- Inspection forms and checklists
- Maintenance procedures
- Operator rounds
- Company logo (optional)
- Site and department structure

---

## Required AssetIQ Configuration

- [ ] Tenant created
- [ ] Modules configured
- [ ] Roles and permissions configured
- [ ] Languages configured
- [ ] Notifications configured
- [ ] Dashboards configured
- [ ] AI configured (if licensed)
- [ ] Mobile application enabled

---

## Go-Live Criteria

A customer is considered ready for go-live when:

1. All required users have access to AssetIQ.
2. Core equipment and maintenance data has been imported and validated.
3. Key business processes have been configured and tested.
4. End users have been trained.
5. Mobile devices are operational.
6. User Acceptance Testing has been successfully completed.
7. The customer has formally approved the go-live.

---

## Summary

This checklist covers the critical factors that determine a successful AssetIQ deployment. Complete it during implementation and revisit items in **Success Readiness** (People, Process, Technology KPIs, assessments, and pulse surveys) to track ongoing adoption after go-live.
